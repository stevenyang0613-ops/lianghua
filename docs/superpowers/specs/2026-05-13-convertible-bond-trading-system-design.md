# 可转债量化交易桌面端系统 — 设计文档

## 1. 项目概述

**项目名称：** LiangHua（量化）

**目标：** 构建一个功能全面的可转债量化交易桌面端 GUI 应用，涵盖实时行情展示、策略回测、实盘交易执行和智能分析工具四大核心模块。对标 GitHub 5000+ star 量化项目（Freqtrade、Superalgos、Jesse、OctoBot 等）的设计水准和工程质量。

**技术栈：** Python FastAPI 后端 + React/Vite/TypeScript 前端 + Electron 桌面壳

**目标用户：** 中国可转债市场的个人量化交易者

### 2.2 进程模型

- **Electron 主进程：** 窗口管理、托盘图标、自动更新、Python 子进程生命周期管理
- **渲染进程：** React SPA，通过 WebSocket + REST 与后端通信
- **Python 子进程：** FastAPI 服务，由 Electron 自动启动和监控

### 2.3 三种运行模式

| 模式 | 数据源 | 交易执行 | 用途 |
|------|-------|---------|------|
| 回测模式 | 历史数据 | 无 | 策略验证和参数优化 |
| 模拟盘模式 | 实时数据 | SimBroker（模拟成交） | 策略上线前验证 |
| 实盘模式 | 实时数据 | 券商 API | 真实交易 |

同一套策略代码在三模式下共享，只需切换 Broker Adapter。

## 3. 模块详细设计

### 3.1 行情模块（Market）

#### 3.1.1 实时行情表

- 展示 300+ 只可转债的实时数据
- 虚拟滚动（react-window），只渲染可见行
- 增量 DOM 更新，价格变化时绿色/红色闪烁动画
- 列定制：拖拽排序、显示/隐藏、列宽持久化
- 条件高亮：溢价率 < 0% 绿色、强赎倒计时 < 15 天红色
- 键盘导航：↑↓ 切换行、Enter 打开详情
- 右键菜单：加入自选/快速下单/查看详情
- 多列排序 + 条件筛选（溢价 > 0、双低 < 150 等）
- 冻结代码 + 名称列

#### 3.1.2 主要展示字段

| 字段 | 说明 | 更新频率 |
|------|------|---------|
| 代码/名称 | 转债代码和简称 | 静态 |
| 最新价 | 当前成交价 | 实时 |
| 涨跌幅 | 当日涨跌幅 | 实时 |
| 正股代码/名称 | 对应正股信息 | 静态 |
| 转股价 | 当前转股价 | 随公告更新 |
| 转股价值 | 正股价 × 100 / 转股价 | 实时 |
| 转股溢价率 | (转债价 - 转股价值) / 转股价值 | 实时 |
| 双低值 | 价格 + 溢价率 × 100 | 实时 |
| 到期收益率(YTM) | 持有到期的年化收益率 | 日频 |
| 剩余年限 | 距到期年限 | 日频 |
| 评级 | AAA/AA+/AA 等 | 静态 |
| 剩余规模 | 未转股余额（亿元） | 日频 |
| 强赎倒计时 | 触发计数及剩余天数 | 随行情 |
| 成交额 | 当日成交额 | 实时 |
| 正股涨跌幅 | 正股当日涨跌幅 | 实时 |

### 3.3 策略回测模块（Backtest）

#### 3.3.1 回测引擎

- 向量化计算（NumPy + Numba JIT）
- 支持 T+0 日内多次交易
- 完整的滑点 + 佣金模型
- 可转债特殊规则模拟：转股、强赎、回售、下修
- 正股 + 转债双价格流联动

#### 3.3.2 回测工作流

```
选策略 → 设参数 → 选时间 → 执行回测 → 查看报告
                                       ↓
                              实时进度推送 (WebSocket)

                              交易列表 (可筛选/排序)
                              月度收益热力图
                              逐笔交易归因
                              策略对比 (多策略叠加)
```

#### 3.3.3 绩效指标

- 累计收益率、年化收益率
- 最大回撤 (Max Drawdown)
- Sharpe Ratio、Sortino Ratio、Calmar Ratio
- 胜率、盈亏比
- 交易次数、平均持有天数
- 月度/年度收益分布
- 收益归因分析（择时/择券贡献）

#### 3.3.4 参数优化

- 网格搜索 + 遗传算法
- 参数热力图可视化
- 3D 曲面展示参数敏感性
- Walking Forward 验证

## 4. 插件系统

统一接口设计，四类插件：

```typescript
interface Plugin {
  id: string;
  name: string;
  version: string;
  type: 'datasource' | 'strategy' | 'broker' | 'notifier';
}

interface DataSourcePlugin extends Plugin {
  type: 'datasource';
  fetch(symbols: string[]): Promise<MarketData>;
}

interface BrokerPlugin extends Plugin {
  type: 'broker';
  buy(order: Order): Promise<OrderResult>;
  sell(order: Order): Promise<OrderResult>;
  getPositions(): Promise<Position[]>;
}
```

内置插件：
- 数据源：AKShare、Tushare、东方财富
- 券商：模拟盘(SimBroker)、QMT、Easytrade
- 通知：桌面通知、Telegram、钉钉/企业微信 Webhook

## 6. 数据流设计

### 6.1 实时行情流

```
外部数据源 ──> 数据采集(定时轮询) ──> 行情引擎(去重/合并) ──> DuckDB(持久化)
                                   │
                                   └──> WebSocket 广播(按订阅主题推送)
                                           │
                                    前端订阅管理(只渲染可见行)
```

WebSocket 协议：
- 订阅：`{"type": "subscribe", "codes": ["123456"]}`
- 取消：`{"type": "unsubscribe", "codes": ["123456"]}`
- 推送：`{"type": "tick", "data": {"code": "113044", "price": 128.5, ...}}`

### 6.2 缓存策略

L1(内存) → L2(DuckDB) → L3(离线可用)

- 断网降级：显示"行情离线"遮罩，回测/分析使用本地缓存
- 后台同步：联网后自动补全缺失数据
- 启动加速：首次全量，后续增量
- 目标：10 秒内可交互

### 6.3 安全设计

| 风险 | 措施 |
|------|------|
| 券商 API Key 明文存储 | Electron safeStorage + 系统钥匙串加密 |
| 策略代码注入 | 沙箱执行 + CPU/内存限制 |
| 数据采集被反爬 | 请求频率控制 + 随机 User-Agent |
| 打包后源码泄露 | PyInstaller 混淆 + 前端 minify |
