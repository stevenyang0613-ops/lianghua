## 1. 策略设计理念

### 🎯 设计目标

```
单一策略的局限:
  - 璇玑: 多因子选股能力强,但回撤控制弱(原 -16.21%)
  - 西部: 风控严格,但换手率极高(原 9704 笔交易)

融合策略目标:
  1. 继承璇玑的"多因子选股能力"
  2. 继承西部的"严格风控"
  3. 通过"交集共识"提高选股确定性
  4. 通过"缓冲带"降低换手率
  5. 通过"追踪止损"控制回撤
```

## 2. 核心融合逻辑

### 🎲 共识选股机制 (Consensus Mechanism)

**核心理念**: 只有两个策略同时看好的标的才是"高质量共识"

```python
# 璇玑初选: 多因子综合得分最高的40只
xuanji_top = set(dd.nlargest(40, 'xj_score')['code'])

# 西部初选: 双低+动量+YTM综合得分最高的30只
xibu_top = set(dd.nlargest(30, 'xb_score')['code'])

# 取交集: 两个策略同时看好
consensus = xuanji_top & xibu_top

# 兜底: 如果交集不足,回退到璇玑TOP
if len(consensus) < 25:
    consensus = xuanji_top

# 在交集内按璇玑得分排序
final_picks = consensus.nlargest(25, 'xj_score')
```

**优势**:
- ✅ 双重验证: 两个独立评分体系互相印证
- ✅ 降低噪音: 单一策略可能误判,共识更可靠
- ✅ 取长补短: 璇玑的"多因子" × 西部的"风险敏感"

### 🛡️ 一票否决制 (Veto Mechanism)

**核心理念**: 任何一条不达标就排除,不留侥幸

| 否决项 | 阈值 | 来源 |
|--------|------|------|
| 信用评分 | < 60 | 价格+评级综合 |
| 溢价率 | > 60% | 西部风控 |
| 强赎倒计时 | < 15天 | 防止末日债 |
| 价格异常 | ≤0 或 >300 | 数据清洗 |

### 🎯 缓冲带机制 (Buffer Mechanism)

```
持仓排序: 1-25名 (核心持仓) + 26-28名 (缓冲带)

调仓规则:
  - 排名1-25: 继续持有
  - 排名26-28 且原持仓: 缓冲观察(连续3天低于25名才卖)
  - 排名>28: 直接卖出
```

**效果**: 减少"擦边进出"导致的不必要交易

### 📉 双重止损体系

```python
# 第一层: 个股追踪止损
if (current_price / peak_price - 1) * 100 <= -5:
    → 卖出该标的

# 第二层: 组合层面止损
if (组合收益率 / 组合最高收益率 - 1) * 100 <= -15:
    → 清空全部持仓
```

    """

#### 3.2 共识选股

```python
# 在 on_data() 中:
xj_scores = self._calc_xuanji_scores(dd)  # 璇玑评分
xb_scores = calc_xb_scores(dd)             # 西部评分

# 双轨排名

xb_top = set(dd.nlargest(30, 'xb_score')['code'])

# 取交集 = 共识
consensus = xj_top & xb_top

# 在共识内再按璇玑得分排序
final = consensus.nlargest(25 + buffer, 'xj_score')
```

#### 3.3 调仓流程

```
每日 on_data() 触发:
  │
  ├─ 组合止损检查 → 触发则全仓卖出
  ├─ 追踪止损检查 → 触发则卖出对应标的
  │
  ├─ 非调仓日 (idx % 7 != 0): return None
  │
  └─ 调仓日:
      ├─ 基础筛选 (价格/溢价率)
      ├─ 一票否决
      ├─ 双轨评分 + 取交集
      ├─ 缓冲带选股
      └─ 生成 buy/sell 信号
```

## 7. 实战建议

### ✅ 适用场景

1. **可投资转债市场**: ≥ 200只可投转债
2. **波动市/震荡市**: 比单边上涨市表现更稳定
3. **资金量**: 100万-1000万 (25只等权持仓,每只4-40万)
4. **交易频率容忍**: 每周可调仓

### ⚠️ 注意事项

1. **建仓期表现偏弱**: 前3个月可能微亏,需耐心持有
2. **追踪止损频繁触发**: 256次/407次 = 62.9%,单笔交易多为短期
3. **数据质量敏感**: PE/PB/行业分类数据质量直接影响评分
4. **调仓成本**: 周频调仓需关注交易手续费

### 🔧 实盘优化方向

1. **增加数据频率**: 日内数据更精确的动量/波动率计算
2. **动态参数**: 根据市场状态自动调整 hold_count / buffer_size
3. **行业轮动**: 加入行业景气度因子
4. **机器学习**: 用XGBoost/LightGBM优化因子权重
5. **多账户分仓**: 大资金分多账户降低冲击成本

### 📊 风险提示

1. **历史回测不代表未来**: 2024-2026转债市场特定环境
2. **极端行情风险**: 单边下跌市可能突破-15%组合止损
3. **流动性风险**: 集中持仓的小盘转债可能流动性不足
4. **政策风险**: 转债新规可能影响策略逻辑
